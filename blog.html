VAR:

  • Function Scope: Variables declared with var are function-scoped. They are only accessible within the function where they are defined.

  • Hoisting: Variables declared with var are hoisted to the top of their containing function or global scope, meaning you can use them before they are declared, but they will be initialized with undefined until their actual declaration.

LET:

  • Block Scope Variables declared with let are block-scoped, meaning they are accessible only within the block where they are defined. A block can be a function, loop, or any enclosed in curly braces{}.

  • Hoisting: Variables declared with let are also hoisted to the top of their containing block, but they are not initialized, so you can't use them before their declaration.

CONST:

  • Block Scope: Like let, const is also block-scoped.

  • Immutability: Variables declared with const are meant to be constants, and their value cannot be reassigned after initialization. However, for objects and arrays declared with const, you can still modify their properties or elements; you just can't reassign the variable itself.

NULL:

  • In JavaScript, null is a special value that represents an empty or unknown value.

  • Usually, null is used to assign 'unknown' or 'empty' value to a variable. Hence, you can assign null to a variable.

UNDEFINED:

  • If a variable is declared but the value is not assigned, then the value of that variable will be undefined.

  • It is also possible to explicitly assign undefined to a variable.

REST API:

  • An API, or application programming interface, is a set of rules that define how applications or devices can connect to and communicate with each other. A REST API is an API that conforms to the design principles of the REST, or representational state transfer architectural style. For this reason, REST APIs are sometimes referred to RESTful APIs.
